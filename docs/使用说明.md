# 在线字幕翻译工具 — 使用说明

本说明分两部分：
- **A. 本地使用**：在自己电脑上跑起来（开发 / 自用）。
- **B. 在线部署使用**：部署到公网服务器（宝塔 Node 项目），多人/远程访问。

两种场景**界面操作完全一致**（见「C. 操作流程」），区别只在「怎么把它跑起来」。

---

## A. 本地使用

### A.1 环境要求

- Node.js ≥ 18（推荐 20 或更高；开发用 24 亦可）
- npm（随 Node 安装）
- 一个可用的 OpenAI 兼容 API（Base URL + Key + 模型名），如 OpenAI / DeepSeek / Kimi / 智谱 / 本地 Ollama

### A.2 启动开发模式

```bash
# 1) 安装依赖（首次）
npm install

# 2) 启动开发服务器
npm run dev
```

浏览器打开 **http://localhost:3000** 即可使用。

### A.3 本地以「生产模式」运行（更接近线上）

```bash
npm run build      # 构建
npm start          # 启动生产服务（默认 3000 端口）
```

如需换端口：

```bash
# macOS / Linux
PORT=4000 npm start
# Windows PowerShell
$env:PORT=4000; npm start
```

### A.4 运行测试（可选，验证核心逻辑）

```bash
npm test           # 跑一遍单测：解析往返一致、行对齐、编码、命名等
```

### A.5 本地使用注意

- 本地直接用 `localhost`，**无需 HTTPS**，API Key 只留在你本机浏览器，安全风险低。
- 若用本地 Ollama：Base URL 填 `http://localhost:11434/v1`，模型名填你 `ollama pull` 的模型（如 `qwen2.5`）。

## B. 在线部署使用（宝塔「Node 项目」）

> 目标形态：Nginx 反代 + Let's Encrypt HTTPS + 网站密码（Basic Auth）把整站挡在登录后。

### B.1 准备

- 一台装好**宝塔面板**的服务器，已安装 **Node 版本管理器** 与 **PM2**（宝塔软件商店里有）。
- 一个解析到服务器的**域名**（用于上 HTTPS）。
- 把本项目代码上传到服务器某目录，如 `/www/wwwroot/subtitle-translator`。

### B.2 构建

在项目目录执行（可用宝塔「终端」或「Node 项目」的命令钩子）：

```bash
npm install
npm run build
```

### B.3 新建「Node 项目」

1. 宝塔左侧「网站」→「Node 项目」→「添加 Node 项目」。
2. **项目目录**：指向 `/www/wwwroot/subtitle-translator`。
3. **启动选项 / 启动命令**：`npm start`（Next.js 默认监听 **3000** 端口）。
4. **运行方式**：PM2（守护进程，崩溃自动重启）。
5. 保存并启动，确认进程在跑、3000 端口已监听。

> 提示：也可使用 standalone 产物（`.next/standalone`）来跑，本项目已开启 `output: "standalone"`。简单起见，直接 `npm start` 即可。

### B.4 反向代理 + HTTPS

1. 「网站」→「添加站点」，绑定你的域名（可不建数据库、纯静态占位）。
2. 进站点设置 →「反向代理」→ 添加：
   - 目标 URL：`http://127.0.0.1:3000`
   - 发送域名：`$host`
3. 站点设置 →「SSL」→ 选「Let's Encrypt」一键签发，开启「强制 HTTPS」。

### B.5 整站加密码（强烈建议）

站点设置 →「网站密码」/「访问限制」→ 设置 **HTTP Basic Auth** 用户名密码。
这样所有人访问前都要先登录，配合 HTTPS 才能安全地在公网用带 Key 的工具。

> ⚠️ 公网 + 带 Key + 没有 HTTPS = 不可接受。务必先 HTTPS 再用。

### B.6 升级 / 重启

代码更新后：

```bash
git pull            # 或重新上传代码
npm install         # 依赖有变动时
npm run build
```

然后在宝塔「Node 项目」里点「重启」。

### B.7 部署后访问

浏览器打开 `https://你的域名` → 输入网站密码 → 进入工具界面。

## C. 界面布局总览

打开后界面分三块：

- **顶部工具栏**（始终可见）：左侧「字幕翻译」标题 + 翻译状态徽标（未翻译 / 翻译中 x% / 已完成翻译）；右侧 —— **打开新文件**、**导出仅译文**、**导出双语**、**主题切换**（🖥跟随系统 / ☀浅色 / 🌙深色，默认跟随系统）、**⚙ 设置**。
- **控制条**（上传文件后出现，吸附在工具栏下）：**翻译服务**切换、**原始语言**、**目标语言**、**开始翻译/取消**按钮。
- **主体**：上传前是整屏拖拽区；上传后是**完整字幕表格**（序号 / 开始时间 / 结束时间 / 原文 / 译文），所有行都显示，**译文一栏可直接点击修改**，翻译过程中实时回填。

所有细项配置都收在 **⚙ 设置**弹窗里，分四个标签页：翻译服务、翻译参数、高级·提示词、双语与导出。
**设置弹窗里的改动是「草稿」，点底部「保存」才生效；点「关闭」则放弃本次改动。**（控制条上的翻译服务/原始语言/目标语言属于快捷切换，即时生效，不走草稿。）

## D. 操作流程（本地与在线通用）

### 第 1 步 · 配置翻译服务（首次必做）
点右上角 **⚙ 设置 →「翻译服务」**标签：
- 点 **「+ 添加服务」**，再点服务预设（**OpenAI / DeepSeek / Moonshot / 智谱 / Ollama**）会**一键带入名称、Base URL、模型名**，按需改 API Key 即可；也可全部手填。
- 点 **「测试连接」** 校验是否可用（绿色为成功）。
- 可添加**多个服务**，用单选按钮选「使用此服务」。**改完点底部「保存」**。之后在控制条的「翻译服务」下拉里可随时切换。

> 密钥只存本地浏览器，经转发路由穿透，不进源码、不进前端包。

### 第 2 步 · 打开字幕
- 上传前的整屏区域里**拖拽或点选** `.srt`；或随时用工具栏 **「打开新文件」**。
- 单文件 ≤ 5 MB，v1 仅支持 SRT。自动探测编码。
- **若表格里是乱码**：上传区下方会出现编码下拉框，手动选 `gbk` / `big5` / `shift_jis` / `utf-16le` 等重新解析。

### 第 3 步 · 选语言、调参数
- 控制条里选 **原始语言**（可「自动检测」）和 **目标语言**。
- 需要时进 **⚙ 设置 →「翻译参数」**：
  - **请求最大段落数**（默认 **20**，即每次请求发给模型的条数）
  - **温度 temperature**（默认 **0**）
  - 并发请求数（默认 6）、最大重试（默认 3）、上下文参考条数（默认 3，提升连贯性）、max tokens
  - 自定义风格指令（追加，可选）
- 进阶可在 **「高级 · 提示词」**标签里改系统提示词模板（支持 `{{to}}`/`{{from}}` 占位符；引擎会自动追加行对齐协议，改风格不会破坏对齐）。可「恢复默认」，也可查看「沉浸式翻译预设」作参考。

### 第 4 步 · 翻译
- 点控制条 **「开始翻译」**，状态徽标显示「翻译中 x%」，表格译文列实时回填。
- 可随时 **「取消」**；已翻译的行保留。
- 中断/失败后按钮变为 **「继续翻译（剩 N）」**——只补译未完成的行（断点续传）。
- 个别多次重试仍失败的行，译文框会标红显示「未翻译」，可**手动填写**或再点「继续翻译」。

### 第 5 步 · 校对（可选）
- 直接在表格 **译文列点击编辑**，失焦即保存。导出时用的就是你改后的内容。

### 第 6 步 · 双语布局与导出设置
进 **⚙ 设置 →「双语与导出」**：
- **双语布局**（选项旁有 **?** 悬停说明）：
  - **双轨**（默认）：译文与原文拆成两条字幕、共用同一时间轴。
  - **单轨**：译文与原文写在同一条字幕的两行里。
  - **仅译文**。
- **语言顺序**：译文在前 / 原文在前。
- **合并多行为单行**（推荐开启，默认开）：字幕原文常因排版被拆成多行，双语叠加后行数会翻倍（如原文 3 行→双语 6 行）影响观看；开启后每种语言压成一行，双语共 2 行，由播放器按宽度自动折行。
- **「仅译文」/「双语」附加文字**：可自定义并保留预设（如 `AI中文` / `AI机翻中文`；`AI中英双语` / `AI机翻中英双语`）。点「自动」恢复按语言自动推断。
- **导出语言码**：默认「自动」（按目标语言推断，简体中文→`chs`），可覆盖。
- **SRT 配色方案**：默认「不分色」；另有暖金/青柠/天蓝/樱粉/纯白等**以译文（中文）为主色调**的方案（需播放器支持 `<font color>` 才生效）。改完记得点「保存」。

### 第 7 步 · 导出下载
直接点工具栏的两个按钮，文件名自动生成：
- **导出仅译文** → `原文件名.AI中文.chs.srt`
- **导出双语** → `原文件名.AI中英双语.chs.srt`（源语言已知时；自动检测时为 `AI中文双语`）

## E. 把字幕放进媒体服务器（Plex / Jellyfin / Emby）

1. 导出的文件名如 `电影名.2024.AI中文.chs.srt`，**基础名要和视频文件同名**（导出时已自动去掉原有语言尾标）。
2. 把字幕文件放到**与视频同一目录**。
3. 让媒体服务器刷新/扫描该影片，外挂字幕轨即会按语言码出现。

> 双语字幕没有标准语言码：用目标语言码（如 `chs`）最稳；自造的 `zh-en` 多数媒体服务器识别不了。

## F. 常见问题（FAQ）

**Q：翻译报「模型 API 返回 401/403」？**
A：API Key 或 Base URL 不对。先用「测试连接」排查；注意 Base URL 通常要带 `/v1`。

**Q：一直 429？**
A：触发了供应商限流。调低「并发请求数」、增大「请求最大段落数」、或换额度更高的账号；引擎已对 429 做指数退避重试。

**Q：译文条数和原文对不上 / 时间轴错位？**
A：本工具用「带 ID 结构化输出 + 条数校验 + 缺失重试」防这个。若仍有少量「未翻译」，点「继续翻译」重试；个别难缠的可减小「请求最大段落数」再试。

**Q：表格里是乱码？**
A：在上传区下方手动选正确编码（GBK/Big5/Shift-JIS/UTF-16）后重新解析。

**Q：翻译太慢，怎么提速？**
A：进 ⚙ 设置 →「翻译参数」调高 **并发请求数**（默认 6），并适当增大 **请求最大段落数**；受模型供应商限流约束，过高可能触发 429。

**Q：想浅色/深色主题？**
A：工具栏右侧三态切换：🖥 跟随系统（默认）/ ☀ 浅色 / 🌙 深色，选择会被记住。

**Q：标签（斜体等）丢了或位置乱了？**
A：整行 `<i>…</i>` 这类会被正确保留；复杂的行内多段标签在译文长度变化大时可能位置近似——v1 已尽量保护，完整精确样式留待 ASS 阶段。

**Q：刷新页面后配置还在吗？**
A：API 配置、翻译参数、样式/导出偏好存在浏览器 localStorage，刷新仍在；但**已上传的字幕和翻译进度不持久化**，需要重新上传。

**Q：换台电脑/换浏览器，Key 还在吗？**
A：不在。Key 只存在你当时输入它的那个浏览器里。
